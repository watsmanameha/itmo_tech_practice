## Описание проекта

Данный проект представляет собой веб-приложение глоссария терминов, посвященных методам и подходам к распознаванию паттернов проектирования в исходном коде программных систем. Проект охватывает классические паттерны Gang of Four (GoF), современные подходы статического анализа кода и методы машинного обучения для автоматического обнаружения архитектурных решений.

## Область исследования

Работа фокусируется на пересечении трех ключевых областей:

### 1. Паттерны проектирования GoF
Классические 23 паттерна, разделенные на три категории:
- **Порождающие** (Creational): Singleton, Factory Method, Abstract Factory, Builder, Prototype
- **Структурные** (Structural): Adapter, Bridge, Composite, Decorator, Facade, Flyweight, Proxy
- **Поведенческие** (Behavioral): Chain of Responsibility, Command, Iterator, Mediator, Memento, Observer, State, Strategy, Template Method, Visitor, Interpreter

### 2. Статический анализ кода
Методы анализа исходного кода без его выполнения:
- Построение Abstract Syntax Tree (AST)
- Анализ зависимостей и связей между компонентами
- Извлечение структурных характеристик и метрик кода
- Идентификация архитектурных паттернов на основе синтаксической структуры

### 3. Machine Learning для распознавания паттернов
Современные подходы с использованием ML/DL:
- Graph Neural Networks (GNN) для анализа графов AST
- Code Embeddings с использованием моделей CodeLlama, DeepSeek Coder
- Feature Extraction из исходного кода
- Семантический анализ и классификация архитектурных решений

## Функциональные возможности

- **Интерактивный глоссарий** - структурированное представление терминов с поиском и фильтрацией по категориям
- **Семантический граф** - визуализация взаимосвязей между понятиями паттернов проектирования, методов анализа и ML-подходов
- **REST API** - программный интерфейс для интеграции с другими системами
- **Современный UI** - адаптивный интерфейс на React + TypeScript

## Содержание глоссария

В рамках исследования в глоссарий включены следующие ключевые термины:

### Паттерны проектирования GoF

- **Singleton** - порождающий паттерн для создания единственного экземпляра класса
- **Abstract Factory** - создание семейств связанных объектов
- **Factory Method** - делегирование создания объектов подклассам
- **Strategy** - инкапсуляция взаимозаменяемых алгоритмов
- **Observer** - механизм подписки на события
- **Decorator** - динамическое добавление функциональности
- **Adapter** - совместимость несовместимых интерфейсов
- **Composite** - древовидная структура объектов

### Методы анализа кода

- **Abstract Syntax Tree (AST)** - древовидное представление синтаксической структуры
- **Static Code Analysis** - анализ без выполнения кода
- **Feature Extraction** - извлечение признаков для ML-моделей

### Machine Learning подходы

- **Graph Neural Network (GNN)** - обработка графовых структур
- **Code Embeddings** - векторные представления кода (CodeLlama, DeepSeek Coder)
- **GoF Design Patterns Recognition** - комплексный подход к распознаванию

## Развертывание

### Локальный запуск с Docker

```bash
# Клонируйте репозиторий
git clone <repository-url>
cd itmo_tech_practice

# Запустите приложение
docker-compose up --build
```

Приложение доступно:

- Фронтенд: http://localhost:3000
- Backend API: http://localhost:8000
- API Документация: http://localhost:8000/docs

### GitHub Pages (статический режим)

Проект поддерживает автоматический деплой на GitHub Pages через GitHub Actions. При push в ветку `main`:

1. Генерируется `glossary.json` из backend
2. Данные копируются в статическую сборку frontend
3. Приложение деплоится на GitHub Pages

Подробнее см. [DEPLOYMENT.md](DEPLOYMENT.md)

## API Endpoints

### Термины

- `GET /api/terms` - Получить все термины (с пагинацией)
- `GET /api/terms/{id}` - Получить термин по ID
- `GET /api/terms/search?q={query}` - Поиск терминов
- `GET /api/terms/category/{category}` - Фильтр по категории
- `GET /api/terms/categories` - Список категорий
- `POST /api/terms` - Создать термин
- `PUT /api/terms/{id}` - Обновить термин
- `DELETE /api/terms/{id}` - Удалить термин

### Система

- `GET /` - Информация об API
- `GET /health` - Проверка работоспособности

Полная интерактивная документация доступна по адресу: <http://localhost:8000/docs>
